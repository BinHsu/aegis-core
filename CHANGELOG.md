## [0.1.0-demo-lan] — 2026-04-22

### Added

- **LAN smoke script** — end-to-end local smoke test (`lan-smoke`) with Qdrant supervisor
  (auto-download, background start, trap cleanup) for forker validation.
- **Engine-side RAG retriever** (Phase 3b Slice 8) — hint panel driven by real vector search;
  fixes "hint panel never fires" regression.
- **Hint broadcast + staff-authored override** (Phase 3c Slice 7) — officers can push manual
  hints that supersede RAG results; broadcast fans out to all viewers.
- **Dynamic corpus dropdown** — `ListCorpora` RPC wires the host UI dropdown to live engine
  corpus list rather than static placeholders.
- **Transcript export** (Phase 3c Slice 5) — host can export full transcript as Markdown or
  JSON via the FileSystem provider.
- **Curated speaker label overrides** (Phase 3c Slice 4) — host can assign role labels
  (`Host`, `Client`, `Colleague`, `Speaker_N`) per ARCH §9.2 privacy-by-design rules.
- **Consent flows** (Phase 3c Slice 3) — audio-processing + transcript two-phase consent UI,
  backed by ADR-0024; consent ledger entry written on acceptance.
- **RAG opt-in** (Phase 3c Slice 2) — host selects a RAG corpus per meeting; empty selection
  runs transcript-only (ADR-0023 "RAG opt-in").
- **FileSystem / Notification / AutoUpdate providers** (Phase 3c Slice 1) — Tauri-compliance
  gate; provider injection pattern separates platform capabilities from React components.
- **Engine `seed` subcommand** (Phase 3b Slice 6) — seeds Qdrant from a corpus directory;
  used by the LAN smoke setup.
- **Gateway /metrics on :8081** (Phase 4d) — Prometheus pull endpoint isolated from the
  :8080 application surface; Alloy/kube-prometheus ServiceMonitor-compatible.
- **Staging Kubernetes manifests** (Phase 4c C-1) — ArgoCD-ready `apps/` layout with
  NetworkPolicy, IRSA annotations, and ldz platform contract alignment.
- **Cosign keyless signing + SBOM attestation** (Phase 4b) — gateway and engine OCI images
  signed via Sigstore; CycloneDX SBOM attached as signed Cosign attestation.
- **Trivy CRITICAL-CVE block + SLSA L3 provenance** (Phase 4b mini-slices 2–3) — CVE gate
  blocks ECR push; `actions/attest-build-provenance` emits in-toto v1 provenance statements.
- **Frontend deploy via S3 + CloudFront** (Phase 4a Slice 5) — React/Vite SPA ships to
  `aegis-app.staging.binhsu.org`; CORS allowlist wired for split-subdomain ADR-0027.
- **C++ engine distroless OCI image + ECR push** (Phase 4a Slice 4) — engine ships as a
  distroless container; pushed to ECR on merge to `main`.
- **Gateway distroless OCI image** (Phase 4a Slice 1) — `rules_oci` + distroless base;
  CycloneDX SBOM generated by `anchore/sbom-action`.

### Changed

- Live transcript window extended from 3 s to 5 s (`kLiveWindowSeconds`).
- Transcript segments stitched until sentence-end punctuation, reducing fragmented lines.
- CI workflows pivot to GitHub Repository Variables for forker-friendly configuration
  (replaces hardcoded bucket / distribution ID values).
- RAG chunker rewritten to markdown-aware segmentation with sentence-aware overlap walk.
- Retriever hints gated on `min_score` + deduplication; viewer error latch cleared on
  new hint arrival.
- `download_models.sh` default flipped to download all non-`PLACEHOLDER` models.

### Fixed

- Retriever stripped leading `##` header from hint suggestion and citation quote.
- LAN live transcript three-layer fix for Incident 14 (audio pipeline + session + fan-out).
- CAS preflight walker + LAN/Cloud model loader convergence (ROADMAP 335).
- Go SDK bumped 1.24.12 → 1.24.13 (CVE-2025-68121 `crypto/tls`).
- `MODULE.bazel.lock` committed and removed from `.gitignore` (Incident 15).
- Deploy manifests aligned with ldz #54 platform contract (`apps/` path, NetworkPolicy).
